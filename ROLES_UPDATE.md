# تحديث نظام الأدوار (Roles System Update)

## التغييرات المطبقة

### 1. توحيد مسميات الأدوار
- **MANAGER** (manager): المدير العام - له صلاحيات كاملة على النظام
- **EMPLOYEE** (employee): الموظف - له صلاحيات محددة حسب المجموعة المخصصة له

تم إزالة دور "ADMIN" لتوحيد النظام وتبسيطه.

### 2. صفحة لوحة تحكم المدير الجديدة
تم إنشاء صفحة مخصصة للمدير فقط في:
- **المسار**: `/dashboard/manager`
- **الميزات**:
  - إحصائيات شاملة (عدد الموظفين، الفعاليات، الضيوف)
  - إجراءات سريعة (إضافة موظف، إدارة الموظفين، الإعدادات، التقارير)
  - عرض النشاط الأخير في النظام
  - واجهة مميزة بتصميم احترافي

### 3. الحماية والصلاحيات

#### الصفحات المحمية (للمدير فقط):
- `/register` - إضافة موظف جديد
- `/dashboard/users` - إدارة الموظفين
- `/dashboard/manager` - لوحة تحكم المدير

#### API Endpoints المحمية:
- `POST /api/v1/employees` - إضافة موظف
- `GET /api/v1/employees` - عرض الموظفين
- `PUT/PATCH/DELETE /api/v1/employees/[id]` - إدارة موظف محدد
- `GET /api/v1/manager/stats` - إحصائيات المدير

### 4. التحديثات على الملفات

#### Middleware (middleware.ts)
- إضافة حماية لصفحة `/dashboard/manager`
- التحقق من دور المستخدم قبل السماح بالوصول

#### Sidebar (components/dashboard/DashboardSidebar.tsx)
- إضافة رابط "لوحة المدير" (يظهر للمدير فقط)
- تحديث صلاحيات العرض لجميع الروابط
- تغيير "المستخدمين" إلى "إدارة الموظفين"

#### صفحة إضافة موظف (components/auth/ModernRegisterForm.tsx)
- إزالة خيار اختيار الدور (ADMIN/EMPLOYEE)
- جميع المستخدمين الجدد يتم إضافتهم كموظفين (EMPLOYEE)
- تبسيط النموذج

#### صفحة إدارة الموظفين (app/dashboard/users/page.tsx)
- تحديث واجهة المستخدم
- إزالة فلتر "مدير" من القائمة
- تحديث الإحصائيات

#### API Routes
- `app/api/v1/users/route.ts`: إزالة إمكانية تحديد الدور عند الإنشاء
- `app/api/v1/employees/[id]/route.ts`: إزالة إمكانية تغيير الدور عند التحديث

### 5. الصلاحيات المتاحة للموظفين

يمكن للمدير تعيين مجموعات صلاحيات للموظفين:
- **event_creator**: منشئ فعاليات
- **contact_manager**: مدير جهات الاتصال
- **invitation_sender**: مرسل الدعوات
- **viewer**: مشاهد فقط
- **checkin_staff**: موظف تسجيل الحضور

أو تخصيص صلاحيات فردية حسب الحاجة.

## كيفية الاستخدام

### للمدير:
1. تسجيل الدخول بحساب المدير
2. الوصول إلى "لوحة المدير" من القائمة الجانبية
3. إضافة موظفين جدد من خلال "إضافة موظف جديد"
4. إدارة الموظفين من "إدارة الموظفين"

### للموظف:
- لا يمكن للموظف الوصول إلى:
  - صفحة إضافة موظفين
  - صفحة إدارة الموظفين
  - لوحة تحكم المدير
- يمكنه الوصول إلى باقي الصفحات حسب صلاحياته

## الأمان

- جميع الصفحات الحساسة محمية على مستوى Middleware
- جميع API endpoints محمية باستخدام `requireManager`
- التحقق من الصلاحيات على مستوى Client-side أيضاً
- إعادة توجيه تلقائية للمستخدمين غير المصرح لهم

## ملاحظات مهمة

- المدير الأول يتم إنشاؤه من خلال script خاص (create-admin.ts)
- لا يمكن تغيير دور المستخدم من EMPLOYEE إلى MANAGER من خلال الواجهة
- جميع الموظفين الجدد يتم إضافتهم بدور EMPLOYEE فقط
- المدير له صلاحيات كاملة على جميع أجزاء النظام
